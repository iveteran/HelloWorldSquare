Xvfb (X Virtual Framebuffer)
是一個在 Linux 上運行的虛擬 X11 顯示伺服器，它可以在沒有物理顯示器、顯卡或輸入設備的伺服器環境下（即無頭模式，headless），模擬完整的圖形界面環境。
它將所有圖形操作在記憶體中進行，常被用於在 CI/CD 管道、Docker 容器或伺服器中自動化測試/運行 GUI 應用程式。

核心功能與優勢：
    無頭環境運行： 允許 Java、瀏覽器或自動化工具等需要 X server 的程式在伺服器上正常運行。
    記憶體處理： 所有畫面繪製都在記憶體中完成，不會顯示在任何螢幕上。
    完全模擬： 支援多種螢幕解析度、色彩深度和 X11 擴展。

安裝與使用 (以 Ubuntu 為例)：
    安裝： sudo apt-get install xvfb
    基本運行： 使用 xvfb-run 命令來運行需要 GUI 的程式，例如：
    xvfb-run --server-args="-screen 0 1024x768x24" ./my_gui_app
    常規啟動： Xvfb :1 -screen 0 1024x768x24 & 啟動一個虛擬顯示螢幕 :1。

常見應用場景：
    自動化測試： 執行 Selenium 瀏覽器測試、GUI 界面自動化測試。
    伺服器渲染： 運行需要 OpenGL 或繪圖功能的應用程式。
    解決無顯示卡需求： 用於沒有硬體顯示的伺服器。
